# AGENTS.md

本 vault 是面向高中数学老师长期使用的 Obsidian 题库，主要服务中国大陆新高考体系，尤其以新高考 I 卷风格为基准。

## 总体原则

1. 不要破坏已有题库结构。
2. 初版不强依赖 Obsidian 社区插件。
3. 所有题目必须尽量保留原始资料溯源链接。
4. 不要把 AI 猜测当作事实；不确定之处要明确标记。
5. 普通可靠转写题不要机械加入待校对中心。
6. 只有识别、标签、答案、图形等存在会影响题目正常使用的问题，才加入 `4. 说明/待校对中心.md`。

## 目录职责

- `1. 原始题目`：保存 PDF、Word、截图、扫描件等原始资料。不要删除原始资料。
- `2. 题库/文档化题库`：一整套试卷、一整份练习、一份校考卷、一份高考卷，均建成“一份资料一篇 Markdown 笔记”。
- `2. 题库/单题题库`：零散单题按 7 大板块追加到对应汇总页，不要每题单独新建笔记。
- `4. 说明/高中数学三级标签体系.md`：维护高中数学三级知识点标签体系。
- `3. 模版`：保存整套试卷、单题、出卷需求、讲义需求模板。
- `5. 讲义与试卷`：保存候选题清单、生成试卷说明、讲义草稿。
- `4. 说明/工作流`：保存入库、图片处理、待校对、出卷 Word 工作流说明。
- `2. 题库/附件`：保存原图、截题图、生成文档等附件。

## 新题入库规则

### 整套试卷或整份练习

- 在 `2. 题库/文档化题库` 中创建一篇 Markdown。
- 一份资料一篇笔记。
- 笔记内部按题号分块。
- 每道题标题必须是清楚的二级标题，例如：`## 第 1 题`。
- 每题必须包含：标签、难度、来源、状态、题干、答案与详解、原始资料链接。
- 答案与详解默认展开显示。
- 如需自测或讲义，可额外提供 `<details>` 折叠版答案。

### 零散单题

- 不要每题单独开笔记。
- 按高中数学 7 大板块追加到 `2. 题库/单题题库` 对应页面。
- 每题同样必须有清楚标题、标签、难度、来源、题干、答案与详解、原始资料链接。

## 标签规则

1. 标签必须优先使用 `4. 说明/高中数学三级标签体系.md` 中已有标签。
2. 标签不要使用 `#知识点` 总前缀，也不要使用斜杠层级标签。
3. 标签应拆成三层独立标签，并从大板块开始，例如：
   - `#函数与导数 #导数应用 #恒成立`
   - `#函数与导数 #导数应用 #零点`
   - `#立体几何 #空间关系 #线面垂直`
   - `#解析几何 #圆锥曲线 #定点定值`
4. `4. 说明/高中数学三级标签体系.md` 使用树状展示标签关系，避免重复显示大板块；题目正文中仍按一行独立标签写完整组合。
5. 一题可以有多个知识点组合；相同大板块标签只需写一次。
6. 如果确实需要新增标签：
   - 在题目旁标记“新增标签待确认”。
   - 同步更新 `4. 说明/高中数学三级标签体系.md`。
   - 在 `4. 说明/待校对中心.md` 添加 `标签确认` 待办。

## 难度规则

使用 `difficulty: 1` 到 `difficulty: 5`：

- 1：基础题，高考选择/填空前段或基础概念题。
- 2：基础偏综合，常规方法直接可解。
- 3：中档题，需要两三个知识点或较完整计算。
- 4：中档偏难/压轴前置，需要较强综合、分类讨论或关键转化。
- 5：压轴题或高难专题题，适合拔高、竞赛化思维或高考压轴训练。

## 图片与图形规则

1. 图形题优先保证可用性。
2. 能转成 LaTeX、Markdown 表格、坐标描述、几何关系描述时，优先转成可编辑形式。
3. 立体几何等重绘成本较高的题，可以优先保留清晰原图，不强制重画。
4. 如果图形较复杂但原图可以清楚截取，直接把原图作为附件嵌入题目正文，不必进入待校对中心。
5. 只有当“没有图就无法做题”且无法自动处理时，才创建图形相关待办。
6. 原始 PDF、Word、截图仍要保留为附件和溯源。

## 待校对中心规则

`4. 说明/待校对中心.md` 只记录需要人工介入的问题。

必须进入待校对中心的情况包括：

- OCR 识别不完整。
- 答案或详解缺失，或 AI 补写但不确定。
- 标签无法确定，或新增标签待确认。
- 难度判断明显不确定。
- 题目依赖图形，但图形无法可靠转成文本或嵌入清晰原图。
- 大卷子 PDF 中题图无法稳定裁切，需要老师人工截图或补图。
- Word 试卷生成时某题缺少必要图形，会影响学生作答。

不要把所有 AI 入库题目都默认加入待校对中心。

## 出卷与 Word 生成规则

1. 当用户提出出卷需求时，先根据标签、难度、来源、题量筛选候选题。
2. 先在 Obsidian 中生成候选题清单，供用户人工确认。
3. 用户确认后，再生成可直接使用的学生版 `.docx` 试卷。
4. 默认只生成学生版，不含答案解析。
5. 学生版应包含标题、学生信息栏、题目编号、题干、必要图形。
6. 如果题库里某题有原图但没有可编辑图形，Word 中可以先嵌入原图，但必须在生成报告中提醒。

## Agent Skill 使用规则

本 vault 可能运行在不同环境中：Codex 桌面版、Codex CLI、Claude Code，或只具备部分 skills 的 Intel Mac。Agent 必须先根据当前环境实际可用能力工作，不要假设所有桌面插件都存在。

### 通用使用原则

1. 当用户任务明确命中某个 skill 的职责，或用户直接点名某个 skill，必须优先使用该 skill。
2. 使用任何 skill 前，必须先读取该 skill 的 `SKILL.md`，并遵循其中的 workflow、依赖和禁止事项。
3. 不要一次性加载无关参考文件；只在 `SKILL.md` 指向且当前任务需要时，再读取对应 references、scripts 或模板。
4. 如果 skill 缺失，必要时说明缺失并使用可用替代方案。
5. 如果外部依赖缺失，例如 `npm`、`pandoc`、`poppler`、`LibreOffice`、Python 包等，不要编造结果；应明确说明缺少什么，以及哪些验证或输出因此无法完成。
6. 第三方 skills 视为可复用工具，不视为事实来源。涉及题目来源、年份、官方答案、政策或网页资料时，仍要保留可追溯链接或原始文件。
7. 能用现成稳定 skill 完成的工作，不要重复造轮子；只有现成 skill 无法覆盖本 vault 的特定规则时，才补充项目级说明或轻量脚本。

### 主要 skills 使用场景

| Skill | 使用场景 | 禁止或不推荐使用场景 |
|---|---|---|
| `anthropics/skills@pdf` / `pdf` | 读取 PDF 文本、表格、图片；渲染 PDF 页面为图片；扫描版 PDF 的 OCR 辅助；从 PDF 页面裁切题图并保存到附件。 | 不用于 Word 模板排版；不把 OCR 或模型识别当作确定事实；裁图不清晰时必须进入待校对中心。 |
| `anthropics/skills@docx` / `docx` | 读取现成 Word 模板；分析 `.docx` 排版；生成学生版 Word 试卷；必要时解包 XML 模仿模板格式。 | 不用于 PDF 抽取；未获老师确认候选题前，不直接生成最终 Word；不要生成含答案解析的学生版，除非用户明确要求。 |
| `defuddle` | 用户提供普通网页 URL，需要抽取正文 Markdown 并保留来源链接。 | 不用于 PDF、Word、本地文件、已是 Markdown 的 URL、需要登录或强动态交互的网站。 |
| `obsidian-markdown` | 创建或编辑 Obsidian 笔记、wikilink、embed、callout、frontmatter/properties、题库 Markdown。 | 不要为了普通 Markdown 机械套用 Obsidian 语法；不要破坏已有内部链接。 |
| `obsidian-cli` | 用户明确要求控制正在运行的 Obsidian，或需要通过 Obsidian CLI 搜索、创建、打开、调试 vault。 | Obsidian 未运行或 CLI 不可用时不要假装可用；普通文件编辑优先直接修改 Markdown 文件。 |
| `obsidian-bases` | 创建或编辑 `.base` 文件，制作 Obsidian Bases 表格、卡片、筛选和公式视图。 | 不用于普通 Markdown 表格；不把 Bases 当作初版必需插件。 |
| `json-canvas` | 创建或编辑 `.canvas` 文件、题库知识图谱、流程图式 Obsidian Canvas。 | 不用于普通流程说明；简单流程优先 Markdown 或 Mermaid。 |
| `find-skills` | 用户要求查找、评估新的 skills，或当前任务明显缺少现成 skill。 | 不用于日常入库、出卷、PDF/Word 处理本身。 |

### PDF 入库规则

处理 PDF 原始资料时，优先顺序如下：

1. 先用 PDF 文本抽取工具读取可复制文本。
2. 如果文本抽取失败或公式、图形严重错乱，则渲染页面图片，再使用多模态识别或 OCR 辅助转写。
3. 对题图、几何图、统计图等，能可靠转成 LaTeX、Markdown 表格或文字描述时优先转写。
4. 不能可靠重画但原图清晰时，从 PDF 页面裁切清晰题图，保存到 `2. 题库/附件/截题图`，并在题目正文嵌入。
5. 如果没有图就无法作答，且无法可靠裁切或图像不清晰，必须加入 `4. 说明/待校对中心.md`。
6. PDF 原文件必须保留在 `1. 原始题目`，题目中必须保留原始资料链接。

### Word 出卷规则

生成 Word 试卷时，优先使用 `docx` skill：

1. 如果用户提供 Word 模板，先读取模板内容和样式，尽量模仿标题、页边距、题号、字体、段落、表格和页眉页脚。
2. 如果没有模板，按新高考 I 卷风格生成清爽、可打印的学生版试卷。
3. 生成最终 `.docx` 前，必须先生成候选题清单并等待老师确认，除非用户明确要求跳过确认。
4. 默认只生成学生版，不含答案解析。
5. 题库中只有原图的题目，可以在 Word 中嵌入原图，但生成报告中必须提醒。
6. 生成后尽量转 PDF 或渲染预览检查排版；如缺少 `LibreOffice`、`pandoc` 等依赖，应说明未能完成预览验证。

### 网页与资料检索规则

1. 用户提供网页 URL 时，优先用 `defuddle` 抽取正文；失败时退回 `curl`、浏览器工具或人工说明。
2. 用户要求联网搜索时，必须保留来源链接，并区分官方来源、题库站、论坛或二手转载。
3. 对高考真题、官方答案、政策、教材版本等，优先找官方或权威来源；无法确认时明确标记“不确定”。
4. 不要把搜索结果直接当题库事实；必须结合原始资料、PDF/Word 文件或可靠链接入库。

## 每次任务完成后的报告

完成入库、整理、筛题或生成试卷后，必须在聊天框输出简短报告：

1. 本次新增或修改了哪些文件。
2. 本次新增了哪些题目。
3. 哪些题目被加入待校对中心，以及原因。
4. 哪些题目使用了原图嵌入。
5. 哪些题目有图形待补充或待重绘。
6. 下一步建议老师优先处理什么。
