# 半自动入库工作流

本流程用于后续 Codex/Claude 在你把 PDF、Word、截图、扫描件放入 `1. 原始题目` 后，半自动整理为可检索、可筛题、可出卷的 Markdown 题库。

## 总原则

- 不破坏原始资料。
- 一整套资料建一篇 Markdown；零散单题追加到 7 大板块单题汇总页。
- 标签优先使用 [[4. 说明/高中数学三级标签体系]]。
- 普通且能可靠转写的文字题不要机械加入待校对中心。
- 只有影响题目正常使用、需要人工介入的问题才进入 [[4. 说明/待校对中心]]。

## 标准流程

当有新的 PDF、Word、截图、扫描件进入 `1. 原始题目` 后，Codex/Claude 应按以下步骤处理：

1. **识别新增原始资料**  
   对比 [[1. 原始题目/来源登记]] 或文件修改时间，找出新增资料。

2. **判断资料类型**  
   - 整套试卷/整份练习：在 `2. 题库/文档化题库` 新建一份资料对应的一篇笔记。
   - 零散单题：追加到 `2. 题库/单题题库` 对应板块汇总页。

3. **提取题目内容**  
   尽量从 PDF、Word、截图中识别题干、选项、答案、详解。

4. **拆分题号**  
   一整套题按 `## 第 1 题`、`## 第 2 题` 拆成标题块，便于标题树与内部链接定位。

5. **数学公式转写**  
   尽量转成 LaTeX，如 `$a_n$`、`$\sin x$`、`$f'(x)$`、`$\frac{1}{2}$`。

6. **图像与图形处理**  
   函数图、几何图、表格、统计图优先转为可编辑形式；不能可靠转换时保留清晰原图。详见 [[4. 说明/工作流/图片与图形处理说明]]。

7. **自动补充标签**  
   每题使用三级知识点标签，例如：
   - #函数与导数 #导数应用 #恒成立
   - #立体几何 #空间关系 #线面垂直
   - #解析几何 #圆锥曲线 #定点定值

8. **估计难度**  
   使用 `difficulty: 1` 到 `difficulty: 5`，定义见 [[4. 说明/高中数学三级标签体系#难度体系]]。

9. **生成或更新题库笔记**  
   按 [[3. 模版/整套试卷入库模板]] 或 [[3. 模版/单题入库模板]] 写入。

10. **判断是否进入待校对中心**  
    普通文字题若题干、公式、答案较可靠，则只标记 `status: 初步入库` 或 `status: 已转写`，不加入待校对中心。

11. **输出处理报告**  
    在聊天框汇报新增/修改文件、新增题目、待校对问题、原图嵌入、图形待补充与下一步建议。

## 普通文字题处理规则

如果题目来自清晰 PDF、Word、可复制文本，且公式、题干、答案能较可靠转写：

- 不需要进入待校对中心。
- 可以标记：

```yaml
status: 初步入库
```

或

```yaml
status: 已转写
```

后续老师人工看题时，如发现问题，再单独要求 Codex/Claude 标记为待校对。

## 必须进入待校对中心的情况

1. OCR 识别不完整，例如题干缺字、公式明显异常、选项残缺。
2. 答案或详解缺失，或由 AI 自行补写且不确定。
3. 标签无法确定，或 Codex/Claude 新增了标签体系中没有的标签。
4. 难度判断明显不确定。
5. 题目依赖图形，但图形无法可靠转成 Markdown、LaTeX 或文字描述。
6. 立体几何、解析几何、函数图像、统计图表等图形题中，原图无法自动截取或无法准确重绘。
7. 原题在一整套大卷子 PDF 里，图片位置不好自动裁切，需要老师人工截图或补图。
8. Word 试卷生成时某题缺少必要图形，可能影响学生作答。

## 待校对写入格式

每条待办必须包含：

- 待办类型：题干校对 / 答案校对 / 标签确认 / 难度确认 / 图形补充 / 图形重绘 / 来源确认
- 具体题目链接：使用 Obsidian 内部链接，能跳转到题目标题
- 当前状态：待处理 / 已完成 / 暂缓
- 简短说明：说明为什么需要人工处理
- 创建日期

示例写法见 [[4. 说明/待校对中心#格式示例]]。
