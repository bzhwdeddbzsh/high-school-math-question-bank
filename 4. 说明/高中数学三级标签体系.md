# 高中数学三级标签体系

> 本标签体系面向中国大陆新高考 I 卷复习、筛题、组卷和讲义编写。标签采用“三层独立标签”：大板块、二级知识点、三级题型或方法分别写成独立标签。

## 使用原则

1. 题目中每个知识点组合写成三层独立标签：`#大板块 #二级知识点 #三级题型或方法`。
2. 标签体系说明文件采用树状展示，避免每一行重复同一个大板块标签。
3. 不再使用斜杠层级标签；不要把“大板块/二级知识点/三级题型”合并成一个标签。
4. 推荐题目写法示例：`#三角函数与解三角形 #三角恒等变换 #诱导公式`。
5. 一题可以有多个知识点组合；相同大板块标签只需写一次。
6. Codex/Claude 自动打标签时，必须优先从本文件选择。
7. 不要随意发明新标签。
8. 如确需新增标签：
   - 在题目旁写明“新增标签待确认”。
   - 同步更新本文件。
   - 在 [[4. 说明/待校对中心]] 添加 `标签确认` 待办。
9. 标签服务于高考复习与检索，不机械照搬教材目录。

## 树状展示如何理解

下面的树状结构表示标签之间的“推荐组合关系”，不是 Obsidian 的斜杠层级标签。

例如本文件中写作：

```markdown
- #数列
  - #等差数列
    - #基本量计算
```

真正给题目打标签时写作：

```markdown
#数列 #等差数列 #基本量计算
```

如果一题同时涉及等差数列性质和前 $n$ 项和，可写作：

```markdown
#数列 #等差数列 #性质应用 #前n项和
```

这样既能用 `#数列` 检索大板块，也能用 `#等差数列` 检索二级知识点，还能用 `#基本量计算`、`#前n项和` 等检索具体题型。

## 难度体系

- `difficulty: 1`：基础题，高考选择/填空前段或基础概念题。
- `difficulty: 2`：基础偏综合，常规方法直接可解。
- `difficulty: 3`：中档题，需要两三个知识点或较完整计算。
- `difficulty: 4`：中档偏难/压轴前置，需要较强综合、分类讨论或关键转化。
- `difficulty: 5`：压轴题或高难专题题，适合拔高、竞赛化思维或高考压轴训练。

## 状态体系

- `待处理`：仅收集，尚未稳定转写。
- `初步入库`：已形成题目块，但尚未人工完整校对。
- `已转写`：题干、答案、标签、难度基本可用。
- `待校对`：存在会影响使用的问题，已进入待校对中心。
- `已校对`：人工确认可用于出卷或讲义。
- `已讲过`：课堂或作业中已经使用过，可用于出卷时排除。

## 未来可选的辅助标签

若后续需要追踪使用历史，可使用少量非知识点标签，例如：

- `#已讲过`
- `#适合周测`
- `#适合月考`
- `#适合讲义例题`
- `#易错题`

这些标签不是知识点标签，不应替代上面的三级知识点标签。
